# rwb.massops — Модуль массовых операций для Bitrix24

Модуль для Bitrix24, позволяющий массово создавать сущности CRM (Компании) из Excel-файлов (.xlsx) с валидацией данных, проверкой дублей и автоматической генерацией CID.

## 1. Требования

- Bitrix24 (on-premise)
- PHP 8.1+
- Расширения PHP: `mbstring`, `ctype`
- Composer (для установки зависимостей)

## 2. Установка и настройка

### 2.1. Скопировать модуль

Скопируйте директорию `rwb.massops` в папку модулей Bitrix:

```
/local/modules/rwb.massops/
```

### 2.2. Установить зависимости

```bash
cd /local/modules/rwb.massops
composer install --no-dev
```

### 2.3. Установить модуль в Bitrix

Перейдите в административную панель:

```
Настройки → Marketplace → Установленные решения
```

Или через PHP:

```php
\Bitrix\Main\ModuleManager::isModuleInstalled('rwb.massops');
// В install/index.php есть класс установки
```

Также можно установить через административный интерфейс по прямому URL:

```
/bitrix/admin/partner_modules.php?id=rwb.massops&install=Y
```

### 2.4. Права доступа

По умолчанию страница модуля `/massops/` доступна только администраторам Bitrix24. Дополнительно можно настроить группу пользователей с доступом через настройки модуля (см. п. 2.5).

### 2.5. Настройки модуля

Панель настроек: **Настройки → Marketplace → Установленные решения → rwb.massops → Настройки**.

| Параметр | По умолчанию | Описание |
|----------|-------------|----------|
| Размер пачки | `50` | Строк за один тик фонового агента |
| Интервал агента | `30` | Секунд между запусками агента |
| Максимальный размер файла | `10` | Лимит загружаемого файла в МБ |
| Код страны по умолчанию | `RU` | ISO 3166-1 alpha-2 для телефонов без явного кода (11-значные номера на 8 → `+7`) |
| Разделитель мультиполей | `,` | Символ-разделитель для полей PHONE и EMAIL |
| Хранить историю задач | `30` | Дней, после которых завершённые и ошибочные задачи (со всеми строками) удаляются агентом очистки |
| Группа доступа | — | Группа пользователей, участники которой получают доступ к модулю наравне с администраторами |

> Операции создания компаний в CRM выполняются в обход стандартных CRM-прав (`disableCheckAccess`). Специальные CRM-права пользователю не нужны — достаточно доступа к самому модулю.

## 3. Настройка CRM

Перед использованием необходимо настроить пользовательские поля компании в CRM.

### Обязательные пользовательские поля компании

| Поле | Тип | XML_ID |
|------|-----|--------|
| ИНН | Строка | `INN` |
| CID | Строка | `CID` |
| Страна | Привязка к элементам инфоблока | `COUNTRY` |

### Инфоблок стран

Инфоблок, привязанный к полю «Страна», должен содержать свойство с кодом `KOD_ALFA_2` — двухбуквенный ISO-код страны (например: `RU`, `AM`, `KZ`).

Это свойство используется для автоматической генерации CID.

> Если любое из этих полей не настроено — модуль сообщит об ошибке конфигурации при попытке импорта.

## 4. Использование

Страница модуля доступна по адресу `/massops/`.

### Шаг 1 — Выбор сущности

Выберите карточку **«Компании»** и нажмите «Далее».

### Шаг 2 — Загрузка файла

1. Скачайте шаблон по ссылке «Скачать шаблон импорта» — в нём содержатся правильные заголовки всех доступных полей.
2. Заполните шаблон данными (со второй строки).
3. Загрузите файл перетаскиванием или нажатием на зону загрузки.
4. Нажмите «Загрузить».

> Поддерживается только формат `.xlsx`. Максимальный размер файла — 10 МБ.

### Шаг 3 — Предпросмотр, проверка и импорт

- **«Проверить»** — валидирует все строки без сохранения в CRM. Показывает ошибки по каждой строке.
- **«Импортировать N записей»** — запускает импорт в фоновом режиме. Строки с ошибками пропускаются.
- **«Очистить и начать заново»** — удаляет данные сессии, возвращает на шаг 1.

После завершения импорта появляется кнопка перехода к списку созданных компаний.

> Данные сессии хранятся 2 часа. Если не выполнить импорт за это время — потребуется загрузить файл заново.

## 5. Форматы данных в файле

| Тип поля | Формат |
|----------|--------|
| Дата | `ДД.ММ.ГГГГ` — например: `15.03.2024` |
| Дата и время | `ДД.ММ.ГГГГ ЧЧ:ММ:СС` — например: `15.03.2024 09:30:00` |
| Целое число | `42`, `-5`, `0` |
| Число с дробной частью | `1500`, `1500.50`, `1500,50` |
| Да/Нет | `Y`, `N`, `1`, `0` |
| Список | Точное значение из CRM |
| Телефон | `+79001234567`, `89001234567`, `+1 (555) 123-45-67` |
| Email | `ivan@example.com` (несколько — через запятую) |
| URL | `https://example.com` (протокол обязателен) |
| Деньги | `15000` или `15000.50\|RUB` |
| Пользователь | ID пользователя или ФИО |

### Поле ИНН

ИНН принимается в любом непустом формате — цифры, буквы и символы допустимы, так как форматы ИНН различаются в зависимости от страны. Пробелы в начале и конце значения автоматически убираются.

### Поле Страна

Числовой ID элемента инфоблока или точное название элемента (например: `5` или `Россия`).

## 6. Как работает модуль

### Архитектура

```
Браузер → PHP (Vue-компонент + API) → Очередь задач → Агент Bitrix → CRM
```

### Процесс импорта

1. **Загрузка файла** — XLSX читается через `phpspreadsheet` (режим `ReadDataOnly`); проверяется структура заголовков.
2. **Сессия** — данные строк сохраняются в сессию Bitrix (ключ `MASSOPS_IMPORT_*`, TTL 2 ч) для предпросмотра и валидации.
3. **Валидация** (`/api/validate`) — каждая строка нормализуется (`RowNormalizer`) и проверяется (`RowValidator`). Проверяются форматы, дубли внутри файла и дубли в CRM по ИНН.
4. **Импорт** (`/api/import`) — создаётся задача в `b_rwb_massops_import_queue`; строки сохраняются постранично в `b_rwb_massops_import_rows` (по 500 за INSERT).
5. **Фоновая обработка** — `ImportAgent` (каждые 30 сек) атомарно захватывает задачу и запускает `ImportProcessor`, который читает из БД только нужный диапазон строк и создаёт компании через `CCrmCompany::add()`.
6. **Генерация CID** — после сохранения компании генерируется CID в формате `{ИНН}-{KOD_ALFA_2}` и записывается в поле `CID` через `CCrmCompany::update()`.
7. **Отчёт** — по завершении задачи доступен XLSX-отчёт (лист «Сводка» + лист «Результаты импорта» с ID, CID и ошибками по строкам).
8. **Очистка** — `CleanupAgent` (раз в сутки) удаляет из обеих таблиц завершённые и ошибочные задачи старше N дней (настройка `Хранить историю задач`, по умолчанию 30).

### Проверка дублей

- **Дубли внутри файла** — если два ИНН в файле совпадают, обе строки получают ошибку.
- **Дубли в CRM** — ИНН из файла сравниваются с существующими компаниями через `CompanyTable::getList()`.

### Структура директорий

```
rwb.massops/
├── install/          — установщик модуля (install/index.php)
├── lib/
│   ├── Import/       — сервисы импорта (ImportService, CompanyImportService)
│   ├── Queue/        — очередь и агенты (ImportAgent, CleanupAgent, ImportProcessor, ImportJobTable, ImportJobRowTable)
│   ├── Service/      — вспомогательные сервисы (CidGenerator, InnDuplicateStrategy, ...)
│   ├── Support/      — хелперы (StatsReportExporter, UserFieldHelper, ...)
│   └── EntityRegistry.php — реестр поддерживаемых сущностей
├── docs/             — пользовательская документация
└── vendor/           — зависимости Composer (не в git)
```
