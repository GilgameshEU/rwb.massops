## 1. Что делает модуль

Модуль позволяет массово создавать **Компании** в CRM Bitrix24 из Excel-файла (.xlsx).

**Процесс работы:**
1. Загружает XLSX-файл с данными компаний
2. Видит предпросмотр данных в таблице
3. Нажимает «Проверить» — система валидирует данные без сохранения
4. Если ошибок нет (или их приемлемое количество) — нажимает «Импортировать»
5. Импорт выполняется в фоне, прогресс отображается в реальном времени

## 3. Шаг 1 — Выбор сущности

На первом шаге отображаются карточки типов CRM-сущностей.

- Нажмите на карточку **«Компании»**
- Карточка выделяется — кнопка **«Далее»** становится активной
- Нажмите «Далее» для перехода к загрузке файла

> Карточки с пометкой **«скоро»** неактивны — эти сущности будут доступны в следующих версиях.

---

## 4. Шаг 2 — Загрузка файла

### Скачивание шаблона

Перед загрузкой рекомендуется скачать шаблон — он содержит все доступные поля выбранной сущности с правильными заголовками колонок.

**Ссылка «Скачать шаблон импорта»** — расположена под зоной загрузки.

> ⚠️ Заголовки колонок в файле должны точно совпадать с заголовками в шаблоне. Система не распознаёт переименованные или произвольно добавленные колонки.

### Загрузка файла

- Перетащите файл в зону загрузки **или** нажмите на неё для выбора файла
- Поддерживаемый формат: **только .xlsx**
- Максимальный размер файла: **10 МБ** (настраивается администратором)

После выбора файла появляется кнопка **«Загрузить»**.

### Что происходит при загрузке

Система немедленно проверяет структуру файла:

| Проверка | Что проверяется |
|----------|----------------|
| Формат файла | Файл является корректным .xlsx |
| Файл не пустой | Есть хотя бы строка заголовка и одна строка данных |
| Заголовки не пустые | Хотя бы одна колонка имеет название |
| Поля существуют в CRM | Каждое название колонки найдено в CRM для выбранной сущности |

Если проверка не прошла — отображается список ошибок, файл не принимается. Исправьте файл и загрузите повторно.

---

## 5. Шаг 3 — Предпросмотр и импорт

После успешной загрузки открывается таблица с данными из файла.

### Кнопка «Проверить»

Выполняет **полную валидацию** данных без сохранения в CRM.

Система проверяет каждую строку: форматы полей, дубликаты внутри файла, дубликаты в CRM. По результатам:
- Строки с ошибками подсвечиваются в таблице
- Появляется список всех ошибок с указанием строк
- Показывается кнопка **«Импортировать N записей»** — если есть строки без ошибок
- Показывается сообщение «Нет записей для добавления» — если все строки с ошибками

> Строки с ошибками **не попадают** в импорт. Только корректные строки будут созданы.

### Кнопка «Импортировать N записей»

Запускает реальный импорт в фоновом режиме.

**Что происходит:**
1. Данные передаются в очередь задач
2. Фоновый агент обрабатывает данные пачками (по 50 строк по умолчанию)
3. Полоса прогресса показывает ход выполнения в реальном времени
4. По завершении отображается итог: сколько записей создано, сколько ошибок

**После завершения** появляется кнопка «Перейти к Компаниям» для просмотра созданных записей.

### Переключатель «Создать кабинеты»

Если включён — при создании компании дополнительно создаётся связанный кабинет (поставщик).

### Кнопка «Очистить и начать заново»

Удаляет загруженные данные из сессии и возвращает на шаг 1. Данные в CRM не затрагиваются.

> ⚠️ Данные в сессии хранятся **2 часа**. Если вы загрузили файл и не выполнили импорт в течение 2 часов — данные будут автоматически удалены, потребуется загрузить файл заново.

## 6. Правила заполнения файла

### Общие правила

- Первая строка файла — **заголовки колонок** (названия полей CRM). Не изменяйте их.
- Со второй строки — данные. Каждая строка = одна запись.
- Пустые строки игнорируются.
- Пустые ячейки в строках данных допустимы — соответствующее поле просто не заполняется.

### Форматы значений по типу поля

#### Дата
```
Формат: ДД.ММ.ГГГГ
Пример: 15.03.2024
```

#### Дата и время
```
Формат: ДД.ММ.ГГГГ ЧЧ:ММ:СС
Пример: 15.03.2024 09:30:00
```
> Секунды обязательны. `15.03.2024 09:30` — неверный формат.

#### Целое число
```
Допустимо: 42, -5, 0
Недопустимо: 3.14, "сто"
```

#### Число с дробной частью
```
Допустимо: 1500, 1500.50, 1500,50 (запятая как разделитель — допустима)
Недопустимо: "тысяча"
```

#### Да/Нет (boolean)
```
Допустимые значения: Y, N, 1, 0, y, n
Y или 1 = Да
N или 0 = Нет
```

#### Список (enum)
Значение должно точно совпадать с одним из допустимых вариантов, как они названы в CRM. При ошибке система покажет список допустимых значений (первые 5).

```
Пример ошибки: Значение «Клиент VIP» не найдено. Допустимые: Клиент, Партнёр, Поставщик...
```

#### Телефон
```
Допустимые форматы:
+79001234567          — международный формат
89001234567           — российский (8 → +7)
+1 (555) 123-45-67    — с разделителями, любой формат

Несколько телефонов: через запятую
Пример: +79001234567, +79009876543
```

> Для телефонов не в российском формате (без `+`) система использует настройку страны по умолчанию (RU). Если ваши данные содержат номера других стран — указывайте их с кодом страны и знаком `+`.

#### Email
```
Один: ivan@example.com
Несколько: ivan@example.com, petr@example.com
```

#### URL
```
Допустимо: https://example.com, http://example.com/page
Недопустимо: example.com (без протокола)
```

#### Денежное значение
```
Формат: СУММА или СУММА|ВАЛЮТА
Пример: 15000
Пример с валютой: 15000.50|RUB
```

#### Пользователь / Сотрудник
```
Допустимо: числовой ID пользователя (например: 12)
           ФИО пользователя (например: Иван Иванов)
```
Если пользователь не найден — строка получит ошибку.

### Поле ИНН

```
Юридическое лицо: 10 цифр (например: 7707083893)
Физическое лицо / ИП: 12 цифр (например: 772012345678)
```

Пробелы и разделители в ИНН допустимы — система уберёт их автоматически.

### Поле Страна

Значение должно соответствовать элементу инфоблока «Страны» — можно указать числовой ID или точное название элемента.

```
Допустимо: 5 (ID элемента) или Россия (название)
```

## 7. Описание ошибок

### Ошибки при загрузке файла

| Ошибка | Причина | Решение |
|--------|---------|---------|
| Файл пустой | Файл не содержит ни одной строки | Добавьте данные |
| Файл содержит только шаблон без данных | Есть только строка заголовков | Добавьте строки с данными |
| Шаблон не содержит ни одного поля | Первая строка пустая | Первая строка должна быть с заголовками |
| В CRM не найдено поле «Название» | Колонка с таким именем не существует | Используйте шаблон; не переименовывайте колонки |
| Файл слишком большой | Превышен лимит размера | Разбейте файл на части |

### Ошибки при валидации строк

| Ошибка | Поле | Решение |
|--------|------|---------|
| Неверный формат даты: ... Ожидается ДД.ММ.ГГГГ | Дата | Исправьте формат, например: `15.03.2024` |
| Неверный формат даты и времени: ... Ожидается ДД.ММ.ГГГГ ЧЧ:ММ:СС | Дата+время | Добавьте секунды: `15.03.2024 09:30:00` |
| Значение «...» не является целым числом | Число целое | Уберите дробную часть или текст |
| Значение «...» не является числом | Число дробное | Используйте только цифры, `.` или `,` |
| Значение «...» недопустимо для поля Да/Нет. Ожидается: Y, N, 1 или 0 | Да/Нет | Используйте `Y`, `N`, `1` или `0` |
| Значение «...» не найдено в списке допустимых. Допустимые: ... | Список | Скопируйте значение точно из списка допустимых |
| Неверный формат телефона: ... | Телефон | Проверьте номер; для зарубежных добавьте `+` и код страны |
| Значение «...» не является корректным URL | URL | Добавьте `https://` в начало |
| Значение «...» не является корректной суммой | Деньги | Используйте формат `1500.00` |
| Пользователь не найден: ... | Пользователь | Проверьте ID или ФИО; пользователь должен существовать в системе |
| Сотрудник не найден: ... | Сотрудник | Аналогично |
| Некорректный формат ИНН «...» | ИНН | 10 цифр для ЮЛ, 12 цифр для ФЛ/ИП |
| Ошибка конфигурации CRM: ... | — | Администратор должен настроить необходимые поля и инфоблок (см. ниже) |

### Ошибки дублирования

| Ошибка | Причина | Решение |
|--------|---------|---------|
| Дубликат ИНН в файле (строки: X, Y) | Две строки в файле имеют одинаковый ИНН | Удалите дубликат из файла |
| Компания с таким ИНН уже существует в CRM (ID: N) | Компания с этим ИНН уже создана в CRM | Удалите строку из файла или обновите запись вручную |

> Проверка на дубликаты по ИНН работает только если в CRM настроено пользовательское поле ИНН с `XML_ID = INN`.

## 8. Автоматическая генерация CID

После успешного создания компании система автоматически формирует и записывает **CID** — уникальный идентификатор компании.

### Формат CID

```
{ИНН}-{код страны}

Примеры:
7707083893-RU
123456789012-AM
```

Код страны — это двухбуквенный ISO-код (`RU`, `AM`, `KZ` и т.д.), который берётся из свойства `KOD_ALFA_2` элемента инфоблока «Страны», привязанного к компании через поле «Страна».

### Условия генерации

CID формируется автоматически, если:
1. В файле заполнены поля **ИНН** и **Страна**
2. Выбранный элемент инфоблока страны содержит непустое свойство `KOD_ALFA_2`

Если ИНН или Страна не указаны в строке — CID для этой компании не генерируется (поле остаётся пустым).

### Требования к настройке CRM

Для работы генерации CID в CRM должны быть настроены:

| Что | Где искать | XML_ID / код |
|-----|-----------|--------------|
| Поле CID компании | Пользовательские поля → Компания | `XML_ID = CID` |
| Поле ИНН компании | Пользовательские поля → Компания | `XML_ID = INN` |
| Поле Страна компании | Пользовательские поля → Компания (тип: Привязка к элементам инфоблока) | `XML_ID = COUNTRY` |
| Свойство инфоблока стран | Инфоблок, привязанный к полю Страна | Код свойства: `KOD_ALFA_2` |

Если хотя бы один из этих элементов не настроен — все строки при проверке и импорте получат ошибку конфигурации, и импорт не будет выполнен. Обратитесь к администратору системы для настройки.

## 9. Статистика и отчёты

### Таб «Статистика»

Показывает историю всех операций импорта в таблице:

| Колонка | Описание |
|---------|----------|
| Пользователь | Кто запустил импорт |
| Сущность | Компании |
| Статус | Ожидает / В обработке / Завершён / Ошибка |
| Всего | Количество строк в файле |
| Успешно | Создано записей |
| Ошибок | Строк с ошибками |
| Создано | Дата создания задачи |
| Начало | Начало обработки |
| Завершено | Дата завершения |
| Отчёт | Кнопка скачивания отчёта |

Кнопка **«Обновить»** — обновляет список.

### Отчёт об ошибках (XLSX)

После нажатия «Проверить» появляется кнопка скачивания отчёта об ошибках. Файл содержит:
- Все строки из исходного файла
- Дополнительную колонку «Ошибки» с описанием ошибок для каждой строки

### Отчёт по задаче импорта (XLSX)

Доступен в таблице статистики по кнопке «Отчёт». Содержит итоговую информацию по конкретной задаче: статус, статистику, ошибки.

## 10. Что тестировать

### Сценарии для проверки

**Базовый флоу:**
- [ ] Скачать шаблон для Компаний, заполнить 5–10 строк корректными данными, загрузить, нажать «Проверить», убедиться что ошибок нет, нажать «Импортировать», дождаться завершения, перейти в список компаний и проверить наличие созданных записей

**Валидация файла:**
- [ ] Загрузить пустой файл → ошибка «Файл пустой»
- [ ] Загрузить файл только с заголовком (без данных) → ошибка «Только шаблон без данных»
- [ ] Загрузить файл с переименованной колонкой → ошибка «В CRM не найдено поле»
- [ ] Загрузить файл размером более 10 МБ → ошибка о превышении размера

**Валидация данных (строки):**
- [ ] Поле даты: значение `01.01.2024` → успешно; `2024-01-01` → ошибка формата
- [ ] Поле даты+времени: `01.01.2024 12:00:00` → успешно; `01.01.2024 12:00` → ошибка
- [ ] Поле списка: корректное значение → успешно; произвольный текст → ошибка с перечнем допустимых
- [ ] Поле Да/Нет: `Y`, `N`, `1`, `0` → успешно; `Да`, `true` → ошибка
- [ ] Поле телефон: `+79001234567` → успешно; `89001234567` → конвертируется в `+79001234567`; `123` → ошибка
- [ ] Поле ИНН: 10 цифр → успешно; 12 цифр → успешно; 9 цифр → ошибка; буквы → ошибка

**Дубликаты:**
- [ ] Два одинаковых ИНН в файле → обе строки помечаются как дубликаты
- [ ] ИНН, уже существующий в CRM → строка помечается как дубликат в CRM

**Частичный импорт:**
- [ ] Файл с 10 строками, 3 из которых с ошибками → после «Проверить» кнопка показывает «Импортировать 7 записей»; после импорта в CRM создаётся ровно 7 записей

**Сессия и очистка:**
- [ ] Загрузить файл, перейти на другую страницу, вернуться → данные сохранились (шаг 3 открывается сразу)
- [ ] Нажать «Очистить и начать заново» → данные удалены, шаг 1

**Статистика:**
- [ ] После импорта перейти на таб «Статистика», убедиться что задача отображается с правильными данными
- [ ] Скачать отчёт по задаче, проверить содержимое файла

**CID (генерация идентификатора):**
- [ ] Строка с заполненными ИНН и Страной → после импорта у компании в поле CID значение вида `7707083893-RU`
- [ ] Строка без Страны → компания создаётся, CID не заполнен
- [ ] Строка без ИНН → компания не создаётся (ИНН обязателен)
- [ ] Нет поля CID (XML_ID=CID) в CRM → все строки получают ошибку «Ошибка конфигурации CRM: Не настроено пользовательское поле CID»
- [ ] Нет свойства KOD_ALFA_2 в инфоблоке стран → все строки получают ошибку конфигурации

**Граничные случаи:**
- [ ] Файл с 1 строкой данных → корректно обрабатывается
- [ ] Все строки с ошибками → сообщение «Нет записей для добавления», кнопка импорта не появляется
- [ ] Пустые ячейки в строках → соответствующие поля не заполняются, строка не считается ошибочной
