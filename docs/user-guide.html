1. Что делает модуль

Модуль позволяет массово создавать Компании в CRM Bitrix24 из Excel-файла (.xlsx).

Процесс работы:

  1. Загружает XLSX-файл с данными компаний
  2. Видит предпросмотр данных в таблице
  3. Нажимает «Проверить» — система валидирует данные без сохранения
  4. Если ошибок нет (или их приемлемое количество) — нажимает «Импортировать»
  5. Импорт выполняется в фоне, прогресс отображается в реальном времени

2. Интерфейс

Страница модуля доступна по адресу /massops/.

Вверху страницы три таба:

  • Массовый импорт — основной функционал
  • Поиск дублей — в разработке (недоступен)
  • Статистика — история всех операций импорта

3. Шаг 1 — Выбор сущности

На первом шаге отображаются карточки типов CRM-сущностей.

  • Нажмите на карточку «Компании»
  • Карточка выделяется — кнопка «Далее» становится активной
  • Нажмите «Далее» для перехода к загрузке файла
Карточки с пометкой «скоро» неактивны — эти сущности будут доступны в следующих версиях.

4. Шаг 2 — Загрузка файла

Скачивание шаблона

Перед загрузкой рекомендуется скачать шаблон — он содержит все доступные поля выбранной сущности с правильными заголовками колонок.

Ссылка «Скачать шаблон импорта» — расположена под зоной загрузки.

⚠️ Заголовки колонок в файле должны точно совпадать с заголовками в шаблоне. Система не распознаёт переименованные или произвольно добавленные колонки.

Загрузка файла

  • Перетащите файл в зону загрузки или нажмите на неё для выбора файла
  • Поддерживаемый формат: только .xlsx
  • Максимальный размер файла: 10 МБ (настраивается администратором)

После выбора файла появляется кнопка «Загрузить».

Что происходит при загрузке

Система немедленно проверяет структуру файла:

ПроверкаЧто проверяется
Формат файлаФайл является корректным .xlsx
Файл не пустойЕсть хотя бы строка заголовка и одна строка данных
Заголовки не пустыеХотя бы одна колонка имеет название
Поля существуют в CRMКаждое название колонки найдено в CRM для выбранной сущности

Если проверка не прошла — отображается список ошибок, файл не принимается. Исправьте файл и загрузите повторно.

5. Шаг 3 — Предпросмотр и импорт

После успешной загрузки открывается таблица с данными из файла.

Кнопка «Проверить»

Выполняет полную валидацию данных без сохранения в CRM.

Система проверяет каждую строку: форматы полей, дубликаты внутри файла, дубликаты в CRM. По результатам:

  • Строки с ошибками подсвечиваются в таблице
  • Появляется список всех ошибок с указанием строк
  • Показывается кнопка «Импортировать N записей» — если есть строки без ошибок
  • Показывается сообщение «Нет записей для добавления» — если все строки с ошибками
Строки с ошибками не попадают в импорт. Только корректные строки будут созданы.

Кнопка «Импортировать N записей»

Запускает реальный импорт в фоновом режиме.

Что происходит:

  1. Данные передаются в очередь задач
  2. Фоновый агент обрабатывает данные пачками (по 50 строк по умолчанию)
  3. Полоса прогресса показывает ход выполнения в реальном времени
  4. По завершении отображается итог: сколько записей создано, сколько ошибок

После завершения появляется кнопка «Перейти к Компаниям» для просмотра созданных записей.

Переключатель «Создать кабинеты»

Если включён — при создании компании дополнительно создаётся связанный кабинет (поставщик).

Кнопка «Очистить и начать заново»

Удаляет загруженные данные из сессии и возвращает на шаг 1. Данные в CRM не затрагиваются.

⚠️ Данные в сессии хранятся 2 часа. Если вы загрузили файл и не выполнили импорт в течение 2 часов — данные будут автоматически удалены, потребуется загрузить файл заново.

6. Правила заполнения файла

Общие правила

  • Первая строка файла — заголовки колонок (названия полей CRM). Не изменяйте их.
  • Со второй строки — данные. Каждая строка = одна запись.
  • Пустые строки игнорируются.
  • Пустые ячейки в строках данных допустимы — соответствующее поле просто не заполняется.

Форматы значений по типу поля

Дата

Формат: ДД.ММ.ГГГГ
Пример: 15.03.2024

Дата и время

Формат: ДД.ММ.ГГГГ ЧЧ:ММ:СС
Пример: 15.03.2024 09:30:00
Секунды обязательны. 15.03.2024 09:30 — неверный формат.

Целое число

Допустимо:   42, -5, 0
Недопустимо: 3.14, "сто"

Число с дробной частью

Допустимо:   1500, 1500.50, 1500,50 (запятая как разделитель — допустима)
Недопустимо: "тысяча"

Да/Нет (boolean)

Допустимые значения: Y, N, 1, 0, y, n
Y или 1 = Да
N или 0 = Нет

Список (enum)

Значение должно точно совпадать с одним из допустимых вариантов, как они названы в CRM. При ошибке система покажет список допустимых значений (первые 5).

Пример ошибки: Значение «Клиент VIP» не найдено. Допустимые: Клиент, Партнёр, Поставщик...

Телефон

+79001234567          — международный формат
89001234567           — российский (8 → +7)
+1 (555) 123-45-67    — с разделителями, любой формат

Несколько телефонов: через запятую
Пример: +79001234567, +79009876543
Для телефонов не в российском формате (без +) система использует настройку страны по умолчанию (RU). Если ваши данные содержат номера других стран — указывайте их с кодом страны и знаком +.

Email

Один:       ivan@example.com
Несколько:  ivan@example.com, petr@example.com

URL

Допустимо:   https://example.com, http://example.com/page
Недопустимо: example.com (без протокола)

Денежное значение

Формат: СУММА или СУММА|ВАЛЮТА
Пример: 15000
Пример с валютой: 15000.50|RUB

Пользователь / Сотрудник

Допустимо: числовой ID пользователя (например: 12)
           ФИО пользователя (например: Иван Иванов)

Если пользователь не найден — строка получит ошибку.

Поле ИНН

ИНН принимается в любом непустом формате — цифры, буквы и символы допустимы, так как форматы ИНН различаются в зависимости от страны.

Примеры:
7707083893        — Россия (ЮЛ, 10 цифр)
772012345678      — Россия (ФЛ/ИП, 12 цифр)
123456789         — другой числовой формат
AB-12345          — буквенно-цифровой формат (другие страны)

Пробелы в начале и конце значения автоматически убираются.

Поле Страна

Значение должно соответствовать элементу инфоблока «Страны» — можно указать числовой ID или точное название элемента.

Допустимо: 5 (ID элемента) или Россия (название)

8. Автоматическая генерация CID

После успешного создания компании система автоматически формирует и записывает CID — уникальный идентификатор компании.

Формат CID

{ИНН}-{код страны}

Примеры:
7707083893-RU
123456789012-AM

Код страны — это двухбуквенный ISO-код (RU, AM, KZ и т.д.), который берётся из свойства KOD_ALFA_2 элемента инфоблока «Страны», привязанного к компании через поле «Страна».

Условия генерации

CID формируется автоматически, если:

  1. В файле заполнены поля ИНН и Страна
  2. Выбранный элемент инфоблока страны содержит непустое свойство KOD_ALFA_2

Если ИНН или Страна не указаны в строке — CID для этой компании не генерируется (поле остаётся пустым).

Требования к настройке CRM

Для работы генерации CID в CRM должны быть настроены:

ЧтоГде искатьXML_ID / код
Поле CID компанииПользовательские поля → КомпанияXML_ID = CID
Поле ИНН компанииПользовательские поля → КомпанияXML_ID = INN
Поле Страна компанииПользовательские поля → Компания (тип: Привязка к элементам инфоблока)XML_ID = COUNTRY
Свойство инфоблока странИнфоблок, привязанный к полю СтранаКод свойства: KOD_ALFA_2
Если хотя бы один из этих элементов не настроен — все строки при проверке и импорте получат ошибку конфигурации, и импорт не будет выполнен. Обратитесь к администратору системы для настройки.

Кнопка «Обновить» — обновляет список.

Отчёт об ошибках (XLSX)

После нажатия «Проверить» появляется кнопка скачивания отчёта об ошибках. Файл содержит:

  • Все строки из исходного файла
  • Дополнительную колонку «Ошибки» с описанием ошибок для каждой строки

Отчёт по задаче импорта (XLSX)

Доступен в таблице статистики по кнопке «Отчёт». Содержит два листа:

  • Сводка — статус, пользователь, статистика (всего / успешно / ошибок), временны́е метки
  • Результаты импорта — все строки из исходного файла с колонками ID, CID и Ошибки
⚠️ Отчёт доступен в течение 30 дней после завершения задачи (настраивается администратором). По истечении этого срока строки удаляются агентом очистки.

10. Настройки модуля

Настройки доступны в административной панели: Настройки → Marketplace → Установленные решения → rwb.massops → Настройки.

Очередь обработки

ПараметрПо умолчаниюОписание
Размер пачки (строк за тик агента)50Количество строк, обрабатываемых фоновым агентом за одну итерацию. Увеличьте при мощном сервере.
Интервал агента (секунды)30Как часто запускается агент. Уменьшите для более быстрой обработки.
Максимальный размер файла (МБ)10Файлы большего размера не принимаются при загрузке.

Телефоны

ПараметрПо умолчаниюОписание
Код страны по умолчаниюRUISO 3166-1 alpha-2 код. Применяется к 11-значным номерам, начинающимся с 8 (например, 89001234567 → +79001234567). Для других стран используйте явный + и код страны.
Разделитель мультиполей,Символ для разделения нескольких значений в полях Телефон и Email (например: +79001234567, +79009876543).

Очистка данных

ПараметрПо умолчаниюОписание
Хранить историю задач (дней)30Завершённые и ошибочные задачи (вместе со строками и возможностью скачать отчёт) хранятся указанное количество дней. По истечении срока агент очистки удаляет их автоматически.
⚠️ После истечения срока хранения отчёт по задаче недоступен. Скачайте отчёт заблаговременно, если данные нужны дольше.

Права доступа

По умолчанию страница модуля доступна только администраторам Bitrix24. Можно дополнительно разрешить доступ участникам конкретной группы пользователей.

ПараметрОписание
Группа пользователей с доступом к модулюВыберите группу — её участники получат доступ к модулю наравне с администраторами. По умолчанию: «— только администраторы —».
Создание компаний в CRM выполняется в обход стандартных CRM-прав — модуль использует disableCheckAccess(). Это значит, что пользователю достаточно иметь доступ к модулю; специальные права CRM на создание компаний не требуются.
